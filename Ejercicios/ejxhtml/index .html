¿Qué es XHTML?

Extensible Hypertext Markup Language (Lenguaje de marcado de hipertexto extensible), HTML expresado como XML, es más estricto a nivel técnico, esto permite que luego sea más fácil al hacer cambios o buscar errores. Su versión 1.0 es la versión XML de HTML, osea que tiene las mismas funcionalidades.


Ventajas de XHTML frente a HTML

-Se pueden incorporar elementos de distintos espacios de nombres XML como MathML (Mathematical Markup Language, lenguaje de marcado) o Scalable Vector Graphics (Gráficos Vectoriales Escalables o Redimensionables, formato de gráficos vectoriales bidimensionales).
-El navegador no necesita implementar estrategias para detectar que quería poner el autor, entonces el parser (analizador sintáctico) puede ser mucho más sencillo.
-Como es XML se pueden utilizar de forma fácil herramientas creadas para procesamiento de documentos XML genéricos como editores, XSLT (Transformaciones XSL), etc.


Diferencias entre XHTML y HTML

-En XHTML para que un documento se muestre en el navegador tiene que cumplir todas las reglas sintácticas, en HTML se pueden mostrar aunque contengan errores sintácticos, su visualización puede ser correcta o incorrecta dependiendo del tipo de errores de página.
-En XHTML no puede haber texto no incluido en alguna etiqueta, en HTML si que puede.
-En XHTML los nombres de elementos y atributos deben escribirse en minúsculas, cuando en HTML no es necesario, lo puedes hacer tanto en minúsculas como mayúsculas.
-En XHTML las referencias a entidades en hexadecimal deben escribirse en minúsculas, en HTML se pueden escribir en minúsculas y mayúsculas.
-Los documentos XHTML tienen que tener un único elemento raíz de todo el documento, el elemento html, los de HTML pueden no tenerlo.
-En XHTML los elementos que tienen etiqueta de apertura deben tener etiqueta de cierre y en HTML no es necesario.
-En XHTML la etiqueta de apertura de los elementos vacíos debe acabar en “ />“ o debe haber una etiqueta de cierre, en HTML no tiene porque.
-En XHTML el elemento pre no puede contener elementos img, object, big, small, sub o sup mientras que en HTML si.
-En XHTML los atributos deben tener siempre valor, HTML pueden no tener valor asignado.
